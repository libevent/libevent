# libevent 系统学习课程大纲

> 共 15 节课，每节课预计 2~3 小时（含阅读源码 + 动手实践）。
> 循序渐进：前 5 课打地基，中间 7 课覆盖全部核心模块，后 3 课进阶与综合。

---

## 第 1 课：前置知识 —— IO 多路复用原理

**学习目标**：理解 libevent 存在的根本原因，掌握其解决的底层问题。

### 核心内容
- 阻塞 IO 与非阻塞 IO 的区别
- `select`：fd_set 的限制（最大 1024 fd）、每次调用需重新注册
- `poll`：突破 fd 数量限制，但仍是线性扫描
- `epoll`：事件驱动、红黑树注册、就绪链表返回；`ET` vs `LT` 模式
- 为什么需要 libevent：跨平台统一抽象 + 事件管理框架

### 实践
```bash
# 手写一个最小 epoll echo server（不使用 libevent）
# 对比感受裸 epoll 与 libevent 的代码量差异
```

### 参考
- `man 2 epoll_create` / `man 2 epoll_wait`
- UNIX Network Programming Vol.1 第 6 章

## 第 3 课：事件循环核心 —— event_base 详解

**学习目标**：深入理解 event_base 的创建、配置和生命周期管理。

### 核心内容

**event_base 的创建与配置**
```c
event_base_new()                    // 默认配置创建
event_base_new_with_config()        // 自定义配置
event_config_new()                  // 创建配置对象
event_config_avoid_method()         // 禁用某个后端
event_config_require_features()     // 要求特定特性（如 ET 模式）
event_config_set_flag()
```

**事件循环控制**
```c
event_base_dispatch()               // 启动（等价于 EVLOOP_ONCE 循环）
event_base_loop(base, flags)        // 细粒度控制
  // EVLOOP_ONCE：处理一次事件后返回
  // EVLOOP_NONBLOCK：不阻塞，立即返回
  // EVLOOP_NO_EXIT_ON_EMPTY：无事件也不退出
event_base_loopexit(base, tv)       // 延迟退出
event_base_loopbreak(base)          // 立即中断
event_base_got_exit() / event_base_got_break()  // 查询退出原因
```

**诊断与调试**
```c
event_base_get_method()             // 查询当前后端名称
event_base_get_features()           // 查询后端特性
event_reinit()                      // fork 后子进程重新初始化
event_base_dump_events()            // 打印所有注册事件（调试用）
```

### 阅读文件
- `include/event2/event.h`（event_base 相关部分）
- `event-internal.h`（`struct event_base` 定义）
- `event.c`（`event_base_new` 实现，重点看后端选择逻辑）

### 实践
```c
// 编写程序：打印当前系统使用的后端，并禁用 epoll 改用 poll
event_config *cfg = event_config_new();
event_config_avoid_method(cfg, "epoll");
struct event_base *base = event_base_new_with_config(cfg);
printf("后端: %s\n", event_base_get_method(base));
```

## 第 4 课：事件管理 —— event 的注册与回调机制

**学习目标**：掌握 event 的完整生命周期，理解事件激活的触发流程。

### 核心内容

**事件标志**
```c
EV_READ       // 可读
EV_WRITE      // 可写
EV_SIGNAL     // 信号
EV_PERSIST    // 持久事件（触发后自动重新注册）
EV_ET         // 边缘触发（需后端支持）
EV_TIMEOUT    // 超时（通常隐式使用）
EV_FINALIZE   // 安全释放标志
EV_CLOSED     // 连接关闭检测
```

**事件生命周期**
```c
event_new(base, fd, events, callback, arg)  // 堆分配
event_assign(ev, base, fd, events, cb, arg) // 栈分配（性能优化）
event_add(ev, timeout)                      // 注册（含可选超时）
event_del(ev)                               // 注销
event_free(ev)                              // 释放（仅堆分配）
event_active(ev, events, ncalls)            // 手动激活
```

**事件状态查询**
```c
event_pending(ev, events, tv)   // 检查是否挂起
event_initialized(ev)
event_get_fd(ev)
event_get_events(ev)
```

**回调函数签名**
```c
void callback(evutil_socket_t fd, short events, void *arg);
// events 可能包含：EV_READ | EV_WRITE | EV_TIMEOUT | EV_SIGNAL
```

### 阅读文件
- `include/event2/event.h`（event 管理 API 部分）
- `event.c`（`event_add`、`event_del`、`event_process_active` 核心逻辑）
- `minheap-internal.h`（定时器的最小堆实现）
- `sample/event-read-fifo.c`（fd 事件完整示例）

### 实践

event_add(ev, new_tv)               // 在回调中重新 add 即可

- 精度与 OS 调度粒度的关系

## 第 6 课：evbuffer —— 高效缓冲区管理

**学习目标**：深入理解 evbuffer 的链式内存设计，掌握零拷贝操作。

### 核心内容

**evbuffer 基本操作**
```c
evbuffer_new() / evbuffer_free()
evbuffer_get_length()               // 可读字节数
evbuffer_add(buf, data, len)        // 追加数据
evbuffer_add_printf(buf, fmt, ...)  // 格式化追加
evbuffer_remove(buf, data, len)     // 读取并移除
evbuffer_copyout(buf, data, len)    // 只读不移除
evbuffer_drain(buf, len)            // 丢弃前 N 字节
```

**高级操作（零拷贝）**
```c
evbuffer_add_reference()            // 引用外部内存（避免拷贝）
evbuffer_add_buffer()               // buffer 间移动（链表操作，O(1)）
evbuffer_reserve_space()            // 预留写空间（scatter/gather IO）
evbuffer_commit_space()
evbuffer_peek()                     // 获取内部 iovec，零拷贝读
```

**面向行/分隔符的读取**
```c
evbuffer_readln(buf, n_read, EOL_STYLE)
// EOL_CRLF / EOL_LF / EOL_NUL / EOL_ANY
evbuffer_search(buf, what, len, start)      // 搜索子串
```

**内部结构理解**
- `evbuffer_chain`：链式内存块，避免频繁 malloc
- 写时拷贝（copy-on-write）的 buffer 引用计数
- 与 fd 的直接 IO：`evbuffer_write(buf, fd)` / `evbuffer_read(buf, fd, n)`

## 第 7 课：bufferevent 基础 —— 异步读写抽象

**学习目标**：掌握 bufferevent 的核心使用模式，理解三类回调的触发时机。

### 核心内容

**bufferevent 核心概念**
- 每个 bufferevent 包含：输入缓冲区（`input`）+ 输出缓冲区（`output`）
- 自动管理：底层 fd 可读时自动读入 input；output 非空时自动写出

**创建与配置**
```c
bufferevent_socket_new(base, fd, options)
// BEV_OPT_CLOSE_ON_FREE：释放时自动关闭 fd
// BEV_OPT_THREADSAFE：线程安全
// BEV_OPT_DEFER_CALLBACKS：延迟回调执行

bufferevent_free(bev)               // 释放（若设了 CLOSE_ON_FREE 同时关 fd）
```

**三类回调**
```c
bufferevent_setcb(bev, readcb, writecb, eventcb, arg)

// readcb：input 中有新数据
void readcb(struct bufferevent *bev, void *arg);

// writecb：output 全部写出（写完通知）
void writecb(struct bufferevent *bev, void *arg);

// eventcb：连接/断开/错误等事件
void eventcb(struct bufferevent *bev, short events, void *arg);
// events: BEV_EVENT_CONNECTED | BEV_EVENT_EOF | BEV_EVENT_ERROR | BEV_EVENT_TIMEOUT
```

**读写操作**
```c
bufferevent_write(bev, data, size)
bufferevent_write_buffer(bev, buf)  // 直接移交 evbuffer
bufferevent_read(bev, data, size)
bufferevent_get_input(bev)          // 获取 input evbuffer（零拷贝读）
bufferevent_get_output(bev)         // 获取 output evbuffer（零拷贝写）
```

**流量控制**
```c
bufferevent_enable(bev, EV_READ | EV_WRITE)
bufferevent_disable(bev, EV_READ)
bufferevent_setwatermark(bev, EV_READ, low, high)   // 读水位控制
// low：input 达到此字节数才触发 readcb（默认 0）
// high：input 超过此字节数时暂停读取（默认无限）
```

**主动发起连接**
```c
bufferevent_socket_connect(bev, addr, addrlen)      // 异步 connect
bufferevent_socket_connect_hostname(bev, dns_base, af, hostname, port)
```

### 阅读文件
- `include/event2/bufferevent.h`（完整阅读）
- `bufferevent.c`（核心抽象层）
- `bufferevent_sock.c`（socket 实现）
- `sample/le-proxy.c`（经典代理示例，重点阅读）

### 实践
```c

// 要求：使用 bufferevent + evconnlistener
```

## 第 15 课：源码精读与综合复盘

**学习目标**：深入理解后端抽象层与 epoll 实现，系统性复盘整个库的设计哲学。

### 核心内容

**后端抽象接口**
```c
// event-internal.h 中的核心结构
struct eventop {
    const char *name;
    void *(*init)(struct event_base *);
    int (*add)(struct event_base *, evutil_socket_t, short, short, void *);
    int (*del)(struct event_base *, evutil_socket_t, short, short, void *);
    int (*dispatch)(struct event_base *, struct timeval *);
    void (*dealloc)(struct event_base *);
    int need_reinit;
    enum event_method_feature features;
    size_t fdinfo_len;
};

// 所有后端按优先级排列
static const struct eventop *eventops[] = {
    &epollops, &pollops, &selectops, ...
};
```

**epoll 后端实现精读**（`epoll.c`）
- `epoll_init`：创建 epoll fd，初始化 changelist
- `epoll_dispatch`：`epoll_wait` 调用 → 遍历就绪事件 → `event_active`
- `epoll_apply_changes`：批量提交 `EPOLL_CTL_ADD/MOD/DEL`（changelist 优化）
- `changelist-internal.h`：变更批处理，减少系统调用次数

**整体事件处理流程（贯穿全书）**
```
event_base_dispatch()
  └─ event_base_loop()
       ├─ timeout_next()          计算最近定时器
       ├─ evsel->dispatch()       等待 IO/信号就绪
       ├─ timeout_process()       处理到期定时器
       └─ event_process_active()  执行所有活跃事件的回调
```

**设计模式总结**
- Reactor 模式：libevent 的核心架构
- 策略模式：`eventop` 接口 + 多后端实现
- 观察者模式：事件注册与回调通知
- 链式 buffer：`evbuffer_chain` 避免大块内存分配
