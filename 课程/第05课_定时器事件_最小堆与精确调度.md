## 1. 定时器是什么

在事件驱动程序中，定时器回答一个简单的问题：**"N 秒之后，请执行这个函数"**。

libevent 的定时器不是操作系统的 `alarm()`，也不是 `setitimer()`，而是完全在用户空间实现的——通过把定时器的到期时间作为 `epoll_wait` 的 `timeout` 参数，巧妙地将 IO 等待和定时器合并到同一个等待调用中：

```
event_base_loop() 每一轮迭代：

  1. 计算最近定时器的剩余时间 → timeout_next()
     ┌────────────────────────────────────────────────────┐
     │  最小堆顶 = 最近要到期的定时器（绝对时间 T）         │
     │  timeout = T - now                                  │
     │  如果已经过期：timeout = 0（立即返回）               │
     │  如果没有定时器：timeout = NULL（永久等待）          │
     └────────────────────────────────────────────────────┘

  2. epoll_wait(epfd, events, maxevents, timeout)
     ← 要么 IO 事件就绪，要么 timeout 到期

  3. 处理到期的定时器 → timeout_process()

  4. 执行活跃事件的回调 → event_process_active()
```

这个设计的精妙之处：不需要任何额外的系统调用，定时器精度与 `epoll_wait` 的精度相同（微秒级，受内核调度影响）。

## 2. 定时器 API

### 2.1 evtimer_* 宏：语义清晰的语法糖

`include/event2/event.h` 第 998~1004 行定义了定时器专用宏，它们只是对 `event_*` 函数的薄封装：

```c
// 等价关系：
#define evtimer_assign(ev, b, cb, arg)  event_assign((ev), (b), -1, 0, (cb), (arg))
#define evtimer_new(b, cb, arg)         event_new((b), -1, 0, (cb), (arg))
#define evtimer_add(ev, tv)             event_add((ev), (tv))
#define evtimer_del(ev)                 event_del(ev)
#define evtimer_pending(ev, tv)         event_pending((ev), EV_TIMEOUT, (tv))
#define evtimer_initialized(ev)         event_initialized(ev)
```

关键参数：
- `fd = -1`：定时器没有文件描述符
- `events = 0`：不监听 IO 事件，只有 timeout 参数触发
- `timeout`：`event_add` 的第二个参数，**相对时间**（距现在多久后触发）

### 2.2 一次性定时器

触发一次后自动删除，不会再触发：

```c
#include <event2/event.h>
#include <stdio.h>

void one_shot_cb(evutil_socket_t fd, short events, void *arg)
{
    printf("3 秒到！\n");
    // 此时 events & EV_TIMEOUT 为真
    // 定时器已被自动 del（没有 EV_PERSIST）
}

int main(void)
{
    struct event_base *base = event_base_new();

    // 方法 1：使用 evtimer_new 宏
    struct event *ev = evtimer_new(base, one_shot_cb, NULL);
    struct timeval three_sec = { 3, 0 };
    evtimer_add(ev, &three_sec);

    // 方法 2：等价的 event_new 写法
    // struct event *ev = event_new(base, -1, 0, one_shot_cb, NULL);
    // event_add(ev, &three_sec);

    event_base_dispatch(base);  // 3 秒后 one_shot_cb 被调用，然后循环无事件自动退出

    evtimer_del(ev);   // 安全起见，即使已触发也可以 del
    event_free(ev);
    event_base_free(base);
    return 0;
}
```

### 2.3 持久定时器（周期性执行）

`EV_PERSIST` 触发后自动重新等待，形成周期性调度：

```c
void periodic_cb(evutil_socket_t fd, short events, void *arg)
{
    struct event_base *base = arg;
    struct timeval now;
    event_base_gettimeofday_cached(base, &now);
    printf("[%ld.%06ld] 周期触发\n", now.tv_sec, now.tv_usec);
    // 不需要再调用 event_add！EV_PERSIST 自动重注册
}

// 创建每 1 秒触发一次的持久定时器
struct event *tick = event_new(base, -1, EV_PERSIST, periodic_cb, base);
struct timeval interval = { 1, 0 };
event_add(tick, &interval);
```

**注意**：`evtimer_new` 不加 `EV_PERSIST`，所以创建持久定时器必须用 `event_new`：

```c
// 错误：evtimer_new 等价于 event_new(base, -1, 0, cb, arg)，没有 EV_PERSIST
struct event *ev = evtimer_new(base, cb, arg);  // ← 只触发一次！

// 正确：手动添加 EV_PERSIST
struct event *ev = event_new(base, -1, EV_PERSIST, cb, arg);
```

### 2.4 在回调中手动重新调度

有时需要在回调中动态决定下次触发时间（不用 PERSIST 的固定间隔）：

```c
void adaptive_cb(evutil_socket_t fd, short events, void *arg)
{
    struct event *ev = arg;   // 通过 event_self_cbarg() 传入

    // 动态计算下次超时（这里模拟随机间隔）
    struct timeval next_delay;
    evutil_timerclear(&next_delay);
    next_delay.tv_sec = 1 + (rand() % 4);  // 1~4 秒随机

    printf("触发，下次在 %ld 秒后\n", next_delay.tv_sec);

    // 手动重新注册（覆盖旧的超时时间）
    event_add(ev, &next_delay);
}

struct event *ev = evtimer_new(base, adaptive_cb, event_self_cbarg());
struct timeval first_delay = { 1, 0 };
event_add(ev, &first_delay);
```

`sample/time-test.c`（第 58~63 行）正是这种模式——非持久定时器在回调中手动 `event_add`：

```c
// time-test.c 第 43~63 行
static void timeout_cb(evutil_socket_t fd, short event, void *arg)
{
    struct timeval newtime, difference;
    struct event *timeout = arg;   // event_self_cbarg() 传入
    double elapsed;

    evutil_gettimeofday(&newtime, NULL);
    evutil_timersub(&newtime, &lasttime, &difference);
    elapsed = difference.tv_sec + (difference.tv_usec / 1.0e6);
    printf("timeout_cb called at %d: %.3f seconds elapsed.\n",
        (int)newtime.tv_sec, elapsed);
    lasttime = newtime;

    if (!event_is_persistent) {
        struct timeval tv;
        evutil_timerclear(&tv);
        tv.tv_sec = 2;
        event_add(timeout, &tv);  // 手动重新调度
    }
}
```

```c
evutil_gettimeofday(&now, NULL);
```

### 3.2 event_base_gettimeofday_cached：高效获取时间

在事件循环内部，libevent 会缓存当前时间（`tv_cache`），避免在同一轮循环中重复调用 `gettimeofday`：

```c
int event_base_gettimeofday_cached(struct event_base *base, struct timeval *tv);
```

`event.c` 第 445 行的实现：

```c
int event_base_gettimeofday_cached(struct event_base *base, struct timeval *tv)
{
    EVBASE_ACQUIRE_LOCK(base, th_base_lock);
    if (base->tv_cache.tv_sec == 0) {
        // 缓存已过期（本轮循环结束时 clear_time_cache() 会清零）
        r = evutil_gettimeofday(tv, NULL);
    } else {
        // 有缓存：时间 = 缓存的单调时间 + 与 wall clock 的偏差
        evutil_timeradd(&base->tv_cache, &base->tv_clock_diff, tv);
        r = 0;
    }
    EVBASE_RELEASE_LOCK(base, th_base_lock);
    return r;
}
```

**使用建议**：在回调函数内需要获取时间时，优先使用 `event_base_gettimeofday_cached`，它比直接调用 `gettimeofday` 快得多（无系统调用，仅内存读取）。

```c
void my_callback(evutil_socket_t fd, short events, void *arg)
{
    struct event_base *base = arg;
    struct timeval now;
    event_base_gettimeofday_cached(base, &now);  // 快！
    // 而不是：gettimeofday(&now, NULL);          // 慢（系统调用）
}
```

## 4. 最小堆：定时器的数据结构

### 4.1 为什么用最小堆

事件循环需要快速回答两个问题：
1. **最近的定时器什么时候到期？**（计算 epoll_wait 超时参数）
2. **哪些定时器已经到期？**（唤醒后处理）

最小堆（Min-Heap）完美满足：
- `O(log N)` 插入（event_add 时）
- `O(log N)` 删除（event_del 时）
- `O(1)` 查看堆顶（最近到期的定时器）
- `O(log N)` 弹出堆顶（处理到期定时器）

### 4.2 min_heap_t 结构

`minheap-internal.h` 第 39~43 行：

```c
typedef struct min_heap {
    struct event **p;  // 指针数组（堆存储在连续内存中）
    size_t n;          // 当前元素数量
    size_t a;          // 已分配容量（a >= n，扩容策略：翻倍）
} min_heap_t;
```

堆用**数组**实现，元素是 `struct event*`。数组中的索引关系：

```
父节点 i 的左子节点：2*i + 1
父节点 i 的右子节点：2*i + 2
子节点 i 的父节点：(i - 1) / 2

比较依据（第 61 行）：
#define min_heap_elem_greater(a, b) \
    (evutil_timercmp(&(a)->ev_timeout, &(b)->ev_timeout, >))

即：ev_timeout（绝对到期时间）小的元素在堆顶
```

每个 event 通过 `ev_timeout_pos.min_heap_idx` 记录自己在堆中的位置，方便 O(log N) 的删除和修改。

### 4.3 堆的可视化

```
假设有 4 个定时器，到期时间分别为：
  ev_A: T+1s   ev_B: T+3s   ev_C: T+2s   ev_D: T+5s

最小堆数组：[ev_A, ev_B, ev_C, ev_D]（索引 0~3）

树形表示：
          ev_A (T+1s)      ← 堆顶，最近到期
         /           \
   ev_B (T+3s)    ev_C (T+2s)
       /
  ev_D (T+5s)

min_heap_top_() 返回 ev_A → timeout_next() 用 T+1s - now 作为 epoll_wait 超时

ev_A 到期后被弹出（min_heap_pop_），堆重建：
          ev_C (T+2s)
         /           \
   ev_B (T+3s)    ev_D (T+5s)
```

### 4.4 核心操作源码解析

**插入（push）**：`minheap-internal.h` 第 71~77 行

```c
int min_heap_push_(min_heap_t* s, struct event* e)
{
    if (min_heap_reserve_(s, s->n + 1))  // 必要时扩容（翻倍策略）
        return -1;
    min_heap_shift_up_(s, s->n++, e);    // 在末尾"开洞"，然后上浮
    return 0;
}
```

**上浮（shift_up）**：把新元素和父节点比较，如果比父节点小就交换，直到满足堆性质：

```c
// minheap-internal.h 第 161~170 行
void min_heap_shift_up_(min_heap_t* s, size_t hole_index, struct event* e)
{
    size_t parent = (hole_index - 1) / 2;
    while (hole_index && min_heap_elem_greater(s->p[parent], e))
    {
        // 父节点比 e 大（到期时间更晚），父节点下移到 hole_index
        (s->p[hole_index] = s->p[parent])->ev_timeout_pos.min_heap_idx = hole_index;
        hole_index = parent;
        parent = (hole_index - 1) / 2;
    }
    // 找到了正确位置，把 e 放进去并记录索引
    (s->p[hole_index] = e)->ev_timeout_pos.min_heap_idx = hole_index;
}
```

**删除（erase）**：`minheap-internal.h` 第 96~115 行

```c
int min_heap_erase_(min_heap_t* s, struct event* e)
{
    if (EV_SIZE_MAX != e->ev_timeout_pos.min_heap_idx) {  // 确认在堆中
        struct event *last = s->p[--s->n];     // 取出最后一个元素
        size_t parent = (e->ev_timeout_pos.min_heap_idx - 1) / 2;

        // 用最后一个元素填补被删除的位置
        // 然后根据情况上浮或下沉（不需要两者都做）
        if (e->ev_timeout_pos.min_heap_idx > 0 &&
            min_heap_elem_greater(s->p[parent], last))
            min_heap_shift_up_unconditional_(s, e->ev_timeout_pos.min_heap_idx, last);
        else
            min_heap_shift_down_(s, e->ev_timeout_pos.min_heap_idx, last);

        e->ev_timeout_pos.min_heap_idx = EV_SIZE_MAX;  // 标记不在堆中
        return 0;
    }
    return -1;
}
```

**扩容策略**：`minheap-internal.h` 第 133~147 行

```c
int min_heap_reserve_(min_heap_t* s, size_t n)
{
    if (s->a < n) {
        size_t a = s->a ? s->a * 2 : 8;  // 初始 8，之后每次翻倍
        if (a < n) a = n;
        s->p = mm_realloc(s->p, a * sizeof *p);
        s->a = a;
    }
    return 0;
}
// 扩容序列：0 → 8 → 16 → 32 → 64 → ...
// 均摊插入复杂度：O(log N)
```

## 5. timeout_next 与 timeout_process

### 5.1 timeout_next：计算 epoll_wait 的超时参数

`event.c` 第 3230~3264 行：

```c
static int timeout_next(struct event_base *base, struct timeval **tv_p)
{
    struct timeval now;
    struct event *ev;
    struct timeval *tv = *tv_p;

    ev = min_heap_top_(&base->timeheap);  // O(1) 查看堆顶

    if (ev == NULL) {
        *tv_p = NULL;   // 没有定时器 → epoll_wait 永久等待
        goto out;
    }

    if (gettime(base, &now) == -1) {  // 获取当前时间（用缓存）
        res = -1;
        goto out;
    }

    if (evutil_timercmp(&ev->ev_timeout, &now, <=)) {
        evutil_timerclear(tv);  // 已经过期 → timeout = 0（立即返回）
        goto out;
    }

    // timeout = 到期时间 - 现在 = 还需要等多久
    evutil_timersub(&ev->ev_timeout, &now, tv);

out:
    return res;
}
```

这个函数确保 `epoll_wait` 的等待时间恰好是到最近定时器到期的剩余时间。

### 5.2 timeout_process：批量处理到期定时器

`event.c` 第 3268~3295 行：

```c
static void timeout_process(struct event_base *base)
{
    struct timeval now;
    struct event *ev;

    if (min_heap_empty_(&base->timeheap)) return;  // 快速检查

    gettime(base, &now);

    // 循环处理所有已到期的定时器（可能有多个同时到期）
    while ((ev = min_heap_top_(&base->timeheap))) {
        // 堆顶的到期时间 > 现在 → 后面的也未到期，停止
        if (evutil_timercmp(&ev->ev_timeout, &now, >))
            break;

        // 从定时器队列删除
        if (!(ev->ev_flags & (EVLIST_ACTIVE|EVLIST_ACTIVE_LATER)))
            event_del_nolock_(ev, EVENT_DEL_NOBLOCK);
        else
            event_queue_remove_timeout(base, ev);

        // 激活：加入活跃队列，传入 EV_TIMEOUT 标志
        event_active_nolock_(ev, EV_TIMEOUT, 1);
    }
}
```

**关键细节**：`timeout_process` 使用 **while 循环**，一次性处理所有到期的定时器。如果系统繁忙导致 `epoll_wait` 多个定时器一起到期，它们都会在这里被激活。

## 6. EV_PERSIST 定时器的时间漂移问题

### 6.1 问题描述

假设注册一个每 1 秒触发的持久定时器，理想情况下触发时刻应该是：
`T0, T0+1, T0+2, T0+3, ...`

但如果每次触发后都用**当前时间 + 1 秒**重新注册，实际触发时刻会是：
`T0, T0+1.001, T0+2.003, T0+3.006, ...`（累积漂移）

因为：
1. `epoll_wait` 唤醒时稍有延迟（内核调度粒度）
2. 回调函数执行需要时间
3. 每次都是从**执行完回调后**的时刻再加 1 秒

### 6.2 libevent 的解决方案：绝对时间对齐

`event.c` 第 1605~1660 行的 `event_persist_closure`：

```c
static void event_persist_closure(struct event_base *base, struct event *ev)
{
    // ev->ev_io_timeout 存放注册时指定的相对间隔（如 1 秒）
    if (ev->ev_io_timeout.tv_sec || ev->ev_io_timeout.tv_usec) {
        struct timeval run_at, now;
        gettime(base, &now);

        // 计算下次触发的绝对时间 = 上次预期时间 + 间隔
        // 而不是 = 当前时间 + 间隔
        evutil_timeradd(&ev->ev_timeout, &ev->ev_io_timeout, &run_at);

        // 安全措施：如果计算出的下次触发时间已经过去了
        // （例如系统非常繁忙，定时器严重延迟）
        // 则用当前时间 + 间隔，避免立即连续触发
        if (evutil_timercmp(&run_at, &now, <)) {
            evutil_timeradd(&now, &ev->ev_io_timeout, &run_at);
        }

        // 用绝对时间重新注册定时器
        event_add_nolock_(ev, &run_at, 1 /* tv_is_absolute */);
    }

    // 执行回调
    (*ev->ev_evcallback.evcb_callback)(
        ev->ev_fd, ev->ev_res, ev->ev_evcallback.evcb_arg);
}
```

**效果**：以上次**预期触发时间**为基准计算下次触发时间，不受回调执行时间的影响，将漂移控制在一个调度周期之内。

### 6.3 对比图

```
非 PERSIST（每次从当前时间重计算）：
  理想：T  T+1  T+2  T+3  T+4
  实际：T  T+1.001  T+2.003  T+3.006  T+4.010  （累积漂移）

EV_PERSIST（从上次预期时间计算）：
  理想：T  T+1  T+2  T+3  T+4
  实际：T  T+1.001  T+2.001  T+3.002  T+4.002  （误差不累积）
```

## 7. common timeout 优化：万个相同超时的场景

### 7.1 问题背景

假设有 10 万个 TCP 连接，每个连接有一个 30 秒的空闲超时（不活跃则断开）。
如果全部插入最小堆：

- 每次连接有活动时，`event_del` + `event_add`：O(log 100000) ≈ 17 次比较
- 堆中 10 万个元素，但超时时间几乎相同（几乎都是"当前时间 + 30s"）

这种情况下最小堆不够高效。libevent 提供了 **common timeout** 优化：对相同超时值的大量事件，用**有序链表**替代最小堆。

### 7.2 原理

```
普通最小堆（10 万个不同超时）：
  插入/删除：O(log N) ≈ 17 次
  查看堆顶：O(1)

common timeout（10 万个相同超时）：
  插入：插入到有序链表末尾，O(1)（因为新事件总是到期时间最晚）
  删除：链表节点删除，O(1)
  查看最近到期：取链表头，O(1)

  原理：相同超时下，后注册的事件总是更晚到期
  → 链表天然有序，无需复杂的堆操作
```

### 7.3 使用方法

```c
// 第一步：为特定超时时长初始化 common timeout
// 返回一个特殊的 "magic" timeval（内部有标记位）
const struct timeval *tv_30s = event_base_init_common_timeout(base, &(struct timeval){30, 0});

// 第二步：使用返回的 magic timeval 注册定时器
// （不能用普通的 {30, 0}，必须用 tv_30s）
event_add(ev1, tv_30s);
event_add(ev2, tv_30s);
event_add(ev3, tv_30s);
// ...10 万个相同超时的事件
```

```
Libevent's default scheduling algorithm is optimized for having a large
number of timeouts with their durations more or less randomly distributed.
But if you have a large number of timeouts that all have the same duration
```

## 8. 定时器精度：实际能有多准？

### 8.1 影响精度的因素

1. **内核调度粒度**：Linux 默认 4ms（CONFIG_HZ=250）或 1ms（CONFIG_HZ=1000）
   - 即使你设置 1ms 超时，实际精度约为 1~4ms

2. **epoll_wait 的精度**：以毫秒为单位，但 Linux 实际精度更高（取决于内核版本）

3. **回调执行时间**：回调本身的耗时会延迟后续定时器

4. **系统负载**：高负载下，进程可能得不到及时调度

5. **单线程限制**：定时器回调在事件循环线程中执行，如果某个回调执行时间很长，会延迟其他定时器

### 8.2 实际测量

```c
// 测量定时器精度的程序
#include <event2/event.h>
#include <event2/util.h>
#include <stdio.h>

static int count = 0;
static struct timeval expected;

void measure_cb(evutil_socket_t fd, short events, void *arg)
{
    struct event_base *base = arg;
    struct timeval now, diff;
    event_base_gettimeofday_cached(base, &now);
    evutil_timersub(&now, &expected, &diff);

    double error_ms = diff.tv_sec * 1000.0 + diff.tv_usec / 1000.0;
    printf("触发 #%d，误差：%.3f ms\n", ++count, error_ms);

    if (count >= 10) {
        event_base_loopbreak(base);
        return;
    }

    // 更新下次预期时间
    struct timeval interval = { 0, 100000 };  // 100ms
    evutil_timeradd(&expected, &interval, &expected);
}

int main(void)
{
    struct event_base *base = event_base_new();
    struct timeval interval = { 0, 100000 };  // 100ms

    event_base_gettimeofday_cached(base, &expected);
    evutil_timeradd(&expected, &interval, &expected);

    struct event *ev = event_new(base, -1, EV_PERSIST, measure_cb, base);
    event_add(ev, &interval);

    event_base_dispatch(base);

    event_free(ev);
    event_base_free(base);
    return 0;
}
// 典型输出（Linux）：
// 触发 #1，误差：0.210 ms
// 触发 #2，误差：0.180 ms
// ...（毫秒级误差）
```

### 8.3 libevent 的单调时钟

为了避免系统时间跳变（NTP 调整、用户修改系统时间）导致定时器异常，libevent 内部使用 **CLOCK_MONOTONIC** 作为计时基准：

`evutil_time.c` 中使用 `clock_gettime(CLOCK_MONOTONIC, ...)` 获取时间，单调递增，不受系统时间修改影响。

`event.c` 第 414~441 行的 `gettime()` 函数：

```c
static int gettime(struct event_base *base, struct timeval *tp)
{
    // 优先使用缓存（同一轮循环内的时间）
    if (base->tv_cache.tv_sec) {
        *tp = base->tv_cache;
        return 0;
    }

    // 使用单调时钟（CLOCK_MONOTONIC）
    if (evutil_gettime_monotonic_(&base->monotonic_timer, tp) == -1)
        return -1;

    // 同时维护 wall clock 的偏差（用于 event_base_gettimeofday_cached 返回正确时间）
    if (base->last_updated_clock_diff + 1 < tp->tv_sec) {
        struct timeval tv;
        evutil_gettimeofday(&tv, NULL);
        evutil_timersub(&tv, tp, &base->tv_clock_diff);
        base->last_updated_clock_diff = tp->tv_sec;
    }
    return 0;
}
```

## 9. event_base_once：一次性定时回调（无需管理事件对象）

有时候你只想"N 秒后执行这个函数"，不需要保存 event 对象：

```c
int event_base_once(struct event_base *base, evutil_socket_t fd,
                    short events, event_callback_fn callback,
                    void *arg, const struct timeval *timeout);
```

```c
// 5 秒后关闭某个连接
event_base_once(base, -1, EV_TIMEOUT, close_connection_cb, conn, &(struct timeval){5, 0});

// 下一轮事件循环立即执行（timeout = 0）
event_base_once(base, -1, EV_TIMEOUT, deferred_init_cb, ctx, &(struct timeval){0, 0});
```

`event_base_once` 内部创建一个临时 event 对象，触发后自动释放，不需要调用者手动 `event_free`。**但要注意**：如果 `event_base_free` 在回调触发前被调用，这个临时 event 会泄漏——使用时需确保 base 的生命周期足够长。

## 10. 完整实战示例

### 10.1 心跳检测服务器

```c
#include <event2/event.h>
#include <event2/listener.h>
#include <event2/bufferevent.h>
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

#define HEARTBEAT_INTERVAL  5   // 5 秒无活动则断开
#define CHECK_INTERVAL      1   // 每 1 秒检查一次

struct connection {
    struct bufferevent *bev;
    struct event       *idle_timer;    // 空闲超时定时器
    struct timeval      last_active;   // 最后活跃时间
    int                 id;
};

static struct event_base *g_base;
static int g_conn_count = 0;

void on_idle_timeout(evutil_socket_t fd, short events, void *arg)
{
    struct connection *conn = arg;
    printf("[连接 %d] 空闲超时，断开\n", conn->id);
    bufferevent_free(conn->bev);
    event_free(conn->idle_timer);
    free(conn);
    g_conn_count--;
}

void reset_idle_timer(struct connection *conn)
{
    // 重置空闲定时器：更新最后活跃时间，重新 add
    event_base_gettimeofday_cached(g_base, &conn->last_active);
    struct timeval timeout = { HEARTBEAT_INTERVAL, 0 };
    event_add(conn->idle_timer, &timeout);
}

void on_read(struct bufferevent *bev, void *arg)
{
    struct connection *conn = arg;
    char buf[256];
    int n;

    while ((n = bufferevent_read(bev, buf, sizeof(buf)-1)) > 0) {
        buf[n] = '\0';
        printf("[连接 %d] 收到: %s", conn->id, buf);
        bufferevent_write(bev, buf, n);  // echo 回去
    }

    reset_idle_timer(conn);  // 有活动，重置超时
}

void on_event(struct bufferevent *bev, short events, void *arg)
{
    struct connection *conn = arg;
    if (events & (BEV_EVENT_EOF | BEV_EVENT_ERROR)) {
        printf("[连接 %d] 关闭\n", conn->id);
        event_free(conn->idle_timer);
        bufferevent_free(bev);
        free(conn);
        g_conn_count--;
    }
}

void on_accept(struct evconnlistener *lev, evutil_socket_t fd,
               struct sockaddr *sa, int salen, void *arg)
{
    struct connection *conn = malloc(sizeof(*conn));
    conn->id = ++g_conn_count;

    conn->bev = bufferevent_socket_new(g_base, fd, BEV_OPT_CLOSE_ON_FREE);
    bufferevent_setcb(conn->bev, on_read, NULL, on_event, conn);
    bufferevent_enable(conn->bev, EV_READ | EV_WRITE);

    // 创建空闲超时定时器（一次性）
    conn->idle_timer = evtimer_new(g_base, on_idle_timeout, conn);
    struct timeval timeout = { HEARTBEAT_INTERVAL, 0 };
    evtimer_add(conn->idle_timer, &timeout);

    printf("[连接 %d] 新连接建立\n", conn->id);
}

int main(void)
{
    g_base = event_base_new();

    struct sockaddr_in sin = {
        .sin_family = AF_INET,
        .sin_port = htons(9999),
    };

    struct evconnlistener *lev = evconnlistener_new_bind(
        g_base, on_accept, NULL,
        LEV_OPT_CLOSE_ON_FREE | LEV_OPT_REUSEABLE, -1,
        (struct sockaddr*)&sin, sizeof(sin));

    printf("服务器启动，端口 9999\n");
    event_base_dispatch(g_base);

    evconnlistener_free(lev);
    event_base_free(g_base);
    return 0;
}
```

### 10.2 带统计的多级定时器

```c
#include <event2/event.h>
#include <event2/util.h>
#include <stdio.h>

// 不同粒度的定时任务
struct timer_task {
    const char       *name;
    int               interval_sec;
    long              run_count;
    struct event     *ev;
    struct event_base *base;
};

void task_callback(evutil_socket_t fd, short events, void *arg)
{
    struct timer_task *task = arg;
    task->run_count++;

    struct timeval now;
    event_base_gettimeofday_cached(task->base, &now);

    printf("[%s] 第 %ld 次执行，时间戳 %ld.%06ld\n",
           task->name, task->run_count, now.tv_sec, now.tv_usec);
}

int main(void)
{
    struct event_base *base = event_base_new();

    // 三个不同频率的定时任务
    struct timer_task tasks[] = {
        { "快速任务 (100ms)", 0, 0, NULL, base },
        { "普通任务 (1s)",    1, 0, NULL, base },
        { "慢速任务 (5s)",    5, 0, NULL, base },
    };
    struct timeval intervals[] = { {0, 100000}, {1, 0}, {5, 0} };

    for (int i = 0; i < 3; i++) {
        tasks[i].ev = event_new(base, -1, EV_PERSIST, task_callback, &tasks[i]);
        event_add(tasks[i].ev, &intervals[i]);
    }

    // 10 秒后退出
    struct event *exit_timer = evtimer_new(base,
        (event_callback_fn)(void*)event_base_loopbreak, base);
    struct timeval ten_sec = { 10, 0 };
    evtimer_add(exit_timer, &ten_sec);

    event_base_dispatch(base);

    for (int i = 0; i < 3; i++) {
        printf("[%s] 共执行 %ld 次\n", tasks[i].name, tasks[i].run_count);
        event_free(tasks[i].ev);
    }
    event_free(exit_timer);
    event_base_free(base);
    return 0;
}
```

## 12. 本课小结

| 概念 | 核心要点 |
|------|---------|
| 定时器实现原理 | 利用 epoll_wait 的 timeout 参数，零额外系统调用 |
| `evtimer_new/add/del` | `event_new(base, -1, 0, cb, arg)` 的语法糖 |
| 一次性 vs 持久 | 无 EV_PERSIST：触发后自动删除；有 EV_PERSIST：自动重注册 |
| `min_heap_t` | 数组实现的最小堆，按 `ev_timeout`（绝对时间）排序 |
| `timeout_next` | 堆顶时间 - 当前时间 = epoll_wait 超时参数 |
| `timeout_process` | while 循环批量处理所有到期定时器，激活后加入活跃队列 |
| EV_PERSIST 漂移修正 | 基于上次预期时间计算下次触发时间，不累积误差 |
| common timeout | 万级相同超时的优化：有序链表代替最小堆 |
| 时间精度 | 受内核调度粒度限制，通常 1~4ms 误差；使用单调时钟避免跳变 |
| `event_base_gettimeofday_cached` | 回调中获取时间的首选，利用缓存避免系统调用 |

**三句话总结本课**：
1. 定时器通过将到期时间转换为 `epoll_wait` 的 timeout 参数实现——**无需额外系统调用**
2. 最小堆保证 O(log N) 的插入/删除和 O(1) 的堆顶查询，是定时器高效的关键数据结构
3. `EV_PERSIST` 定时器使用**绝对时间对齐**而非"当前时间 + 间隔"，有效抑制累积漂移

## 附：练习 4 答案

1. **为什么用最后一个元素填补，而不是上移子节点？**
   上移子节点的方式需要将被删节点以下的所有元素上移，最坏情况是 O(N)。用最后一个元素填补，只需要对这一个元素做 O(log N) 的上浮或下沉调整。

2. **timeout_process 为什么用 while 循环？**
   当系统繁忙时，`epoll_wait` 可能延迟唤醒，导致多个定时器同时到期。while 循环确保一次性处理所有已到期的定时器，不会遗漏。

3. **ev_io_timeout 和 ev_timeout 的区别？**
   `ev_io_timeout` 存储注册时指定的**相对间隔**（如 1 秒），在 `event_persist_closure` 中用于计算下次触发时间；`ev_timeout` 存储每次注册时计算的**绝对到期时间**（now + interval），用于最小堆的排序比较。

4. **在持久定时器回调中调用 event_del 会发生什么？**
   此时定时器已经从最小堆中取出（在 `timeout_process` 中），`ev_flags` 中的 `EVLIST_TIMEOUT` 已被清除，`EVLIST_ACTIVE` 被设置。`event_del` 会从活跃队列中移除（将 `ev_res` 置零），阻止 `event_persist_closure` 的重新注册逻辑。但实际上，`event_persist_closure` 在 `event_del` 之后才执行，需要注意并发场景下的顺序。更安全的做法是在回调中检查条件后才 del。
