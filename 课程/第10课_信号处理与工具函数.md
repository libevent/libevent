## 1. Unix 信号的本质困境

Unix 信号是异步的——它可以在任意时刻打断任何代码（包括 libevent 内部代码）。这带来了一个根本性问题：

```
线程正在执行 event_process_active()
    ↓
SIGINT 到来，内核打断执行
    ↓
跳转到 signal handler
    ↓
你在 handler 里调用了 printf / malloc / event_add...
    ↓
可能死锁、数据损坏（这些函数不是 async-signal-safe 的）
```

**POSIX 规定**：信号处理函数（signal handler）只能调用 async-signal-safe 函数，而 `malloc`、`printf`、`event_add` 等都**不是** async-signal-safe 的。

libevent 的解决方案：**管道技巧（self-pipe trick）**——把信号转换成 IO 事件。

## 2. 管道技巧：信号 → IO 事件

### 2.1 设计原理

```
信号处理流程（signal.c 实现）

传统问题：
  signal_handler() 里不能做复杂操作

libevent 的解法：
  1. 创建一对 socketpair（pipe） → [read_fd, write_fd]
  2. 注册 EV_READ|EV_PERSIST 事件监听 read_fd
  3. signal_handler 只做一件事：向 write_fd 写一个字节（信号编号）
  4. 主循环的 epoll_wait 检测到 read_fd 可读
  5. evsig_cb 读取字节，调用 evmap_signal_active_ 激活信号事件
  6. 用户的信号回调在事件循环中安全地被调用
```

### 2.2 内部数据结构（`evsignal-internal.h` 第 39~62 行）

```c
struct evsig_info {
    /* 监听 socketpair[0] 的内部 event（read_fd 端）*/
    struct event ev_signal;

    /* socketpair：[0]=read端, [1]=write端 */
    evutil_socket_t ev_signal_pair[2];

    /* ev_signal 是否已经 event_add 了 */
    int ev_signal_added;

    /* 当前正在监听的信号数量 */
    int ev_n_signals_added;

#ifdef EVENT__HAVE_SYS_SIGNALFD_H
    /* signalfd 实现：每个信号号对应一个 EV_READ event */
    struct event *ev_sigevent[NSIG];
#endif

    /* 保存被替换前的原始信号处理函数（用于 del 时恢复）*/
    struct sigaction **sh_old;
    int sh_old_max;
};
```

### 2.3 初始化流程（`signal.c` 第 175~209 行）

```c
int evsig_init_(struct event_base *base)
{
    // 1. 创建 socketpair（内部 pipe 实现）
    evutil_make_internal_pipe_(base->sig.ev_signal_pair);
    //    [0] = read fd（事件循环监听这个）
    //    [1] = write fd（signal handler 写这个）

    // 2. 注册内部 event，监听 read_fd 可读
    event_assign(&base->sig.ev_signal, base,
        base->sig.ev_signal_pair[0],
        EV_READ | EV_PERSIST,
        evsig_cb, base);

    // 3. 设为内部事件（不对外暴露）+ 最高优先级
    base->sig.ev_signal.ev_flags |= EVLIST_INTERNAL;
    event_priority_set(&base->sig.ev_signal, 0);

    base->evsigsel = &evsigops;
    return 0;
}
```

### 2.4 信号处理函数（`signal.c` 第 387~437 行）

```c
// 这是真正的 signal handler（系统信号发生时被调用）
// 必须只使用 async-signal-safe 的操作！
static void __cdecl evsig_handler(int sig)
{
    int save_errno = errno;    // 保存 errno（write 会修改它）
    ev_uint8_t msg = sig;      // 把信号编号转成 1 字节

    // 唯一操作：向 write_fd 写 1 字节（write 是 async-signal-safe 的）
    for (;;) {
        errno = EAGAIN;
        if (0 >= write(evsig_base_fd, &msg, 1)) {
            if (errno == EINTR) continue;  // 被另一个信号打断，重试
            // 写失败（管道满了）：信号可能丢失，记录警告
            event_warnx("%s: write: %s", __func__, strerror(errno));
        }
        break;
    }

    errno = save_errno;    // 恢复 errno
}
```

### 2.5 事件循环中读取信号（`signal.c` 第 131~173 行）

```c
// evsig_cb：在事件循环里被调用（安全环境，可以做任何操作）
static void evsig_cb(evutil_socket_t fd, short what, void *arg)
{
    static char signals[1024];
    int ncaught[NSIG] = {0};

    // 循环读取，直到管道为空
    while (1) {
        ssize_t n = read(fd, signals, sizeof(signals));
        if (n == -1) break;         // EAGAIN：读完了
        if (n == 0)  break;

        // 统计每个信号被触发了多少次
        for (int i = 0; i < n; ++i) {
            ev_uint8_t sig = signals[i];
            if (sig < NSIG) ncaught[sig]++;
        }
    }

    // 激活对应的信号事件（安全！在事件循环中执行）
    EVBASE_ACQUIRE_LOCK(base, th_base_lock);
    for (int i = 0; i < NSIG; ++i) {
        if (ncaught[i])
            evmap_signal_active_(base, i, ncaught[i]);
    }
    EVBASE_RELEASE_LOCK(base, th_base_lock);
}
```

**完整数据流**：

```
① 信号发生（SIGINT）
        ↓
② evsig_handler(SIGINT)
   write(write_fd, "\x02", 1)   // SIGINT = 2
        ↓
③ epoll_wait 检测到 read_fd 可读
        ↓
④ evsig_cb 读取字节，ncaught[2]++
        ↓
⑤ evmap_signal_active_(base, 2, 1)
        ↓
⑥ 你的 signal_cb(fd=2, events=EV_SIGNAL, arg) 被调用
```

### 2.6 信号处理的重要限制（`signal.c` 第 67~83 行注释）

```
libevent 的 signal.c 实现有一个历史限制：
同一时刻只有一个 event_base 能接收信号。

如果你向 event_base A 注册了信号，
又向 event_base B 注册了同一个信号，
B 会收到信号，A 不再收到。
（最后注册的 base 获得信号）

对于大多数应用（单个 event_base），这不是问题。
多线程 one-loop-per-thread 架构中，信号只应注册在主线程的 base 上。
```

## 3. signalfd：Linux 的现代方案

Linux 2.6.22+ 提供了 `signalfd(2)` 系统调用，将信号转化为可读的 fd，比 socketpair 更干净。

### 3.1 signalfd 实现（`signalfd.c` 第 52~66 行）

```c
// signalfd 回调：通过 fd 读取信号信息（比 pipe trick 更直接）
static void sigfd_cb(evutil_socket_t fd, short what, void *arg)
{
    struct signalfd_siginfo fdsi;
    struct event_base *base = arg;

    // 从 signalfd 读取结构化的信号信息
    ssize_t ret = read(fd, &fdsi, sizeof(fdsi));

    EVUTIL_ASSERT(ret == sizeof(fdsi));

    EVBASE_ACQUIRE_LOCK(base, th_base_lock);
    evmap_signal_active_(base, fdsi.ssi_signo, 1);
    EVBASE_RELEASE_LOCK(base, th_base_lock);
}
```

每个信号号创建一个独立的 `signalfd` fd，注册为 `EV_READ` 事件。读到数据就说明信号发生了，无需 socketpair。

**signalfd vs 管道技巧的对比**：

| 对比项 | 管道技巧（signal.c）| signalfd（signalfd.c）|
|--------|--------------------|-----------------------|
| 系统调用 | socketpair + write + read | signalfd + read |
| 信号信息 | 只有信号编号 | 完整 signalfd_siginfo（含 PID、UID 等）|
| 多信号共享 | 一个 pipe，按字节编码 | 每个信号一个 fd |
| 可用性 | 所有 Unix | Linux 2.6.22+ |
| 多 base 限制 | 有（全局变量）| 无（每个 base 独立）|

libevent 在编译时检测 `<sys/signalfd.h>`，如果存在则优先使用 signalfd。

## 4. evsignal API 详解

### 4.1 核心宏（`include/event2/event.h` 第 1013~1020 行）

```c
// evsignal_* 都是宏，最终调用 event_* 函数

// 创建信号事件（堆分配）
// 等价于 event_new(base, signum, EV_SIGNAL|EV_PERSIST, cb, arg)
#define evsignal_new(b, x, cb, arg) \
    event_new((b), (x), EV_SIGNAL|EV_PERSIST, (cb), (arg))

// 注册信号事件（无超时）
#define evsignal_add(ev, tv)        event_add((ev), (tv))

// 注销信号事件
#define evsignal_del(ev)            event_del(ev)

// 查询信号事件是否挂起
#define evsignal_pending(ev, tv)    event_pending((ev), EV_SIGNAL, (tv))

// 注意：evsignal_new 总是带 EV_PERSIST
// 信号事件几乎总是持久的（否则只处理一次就失效了）
```

**注意 `EV_PERSIST` 的意义**：信号事件默认带 `EV_PERSIST`。如果不加 `EV_PERSIST`，信号触发一次后事件自动删除，之后的信号就不会再被处理。通常只有想"只处理一次信号"时才去掉 `EV_PERSIST`。

### 4.2 信号回调签名

```c
// 信号回调和普通 event 回调签名相同
void signal_cb(evutil_socket_t sig, short events, void *arg);
// sig：信号编号（如 SIGINT=2, SIGTERM=15）
// events：EV_SIGNAL
// arg：注册时传入的参数

// 获取信号编号的便捷宏
int signum = event_get_signal(ev);  // 等价于 (int)event_get_fd(ev)
```

### 4.3 evsignal_assign：栈分配版

```c
// 避免堆分配（用于性能敏感或嵌入到其他结构中）
struct event sigev;
evsignal_assign(&sigev, base, SIGTERM, signal_cb, arg);
// 等价于 event_assign(&sigev, base, SIGTERM, EV_SIGNAL|EV_PERSIST, cb, arg)
event_add(&sigev, NULL);
```

## 5. 信号处理完整示例

### 5.1 优雅退出（SIGINT/SIGTERM）

```c
// graceful_shutdown.c：收到信号后优雅退出

#include <event2/event.h>
#include <event2/bufferevent.h>
#include <event2/listener.h>
#include <signal.h>
#include <stdio.h>

struct server_stats {
    int connections_total;
    int connections_active;
    long bytes_received;
    long bytes_sent;
};

static struct server_stats g_stats = {0};

static void print_stats(void) {
    printf("\n=== Server Statistics ===\n");
    printf("Total connections:  %d\n", g_stats.connections_total);
    printf("Active connections: %d\n", g_stats.connections_active);
    printf("Bytes received:     %ld\n", g_stats.bytes_received);
    printf("Bytes sent:         %ld\n", g_stats.bytes_sent);
}

// SIGINT：延迟 2 秒退出，给现有连接时间完成
static void sigint_cb(evutil_socket_t sig, short events, void *arg)
{
    struct event_base *base = arg;
    struct timeval delay = {2, 0};

    printf("\nReceived SIGINT, shutting down in 2 seconds...\n");
    print_stats();

    event_base_loopexit(base, &delay);
}

// SIGTERM：立即退出
static void sigterm_cb(evutil_socket_t sig, short events, void *arg)
{
    struct event_base *base = arg;

    printf("\nReceived SIGTERM, shutting down immediately.\n");
    print_stats();

    event_base_loopbreak(base);
}

// SIGUSR1：打印统计信息，继续运行
static void sigusr1_cb(evutil_socket_t sig, short events, void *arg)
{
    printf("\nReceived SIGUSR1 (stats request):\n");
    print_stats();
    // 不退出，继续运行
}

// SIGHUP：重新加载配置
static void sighup_cb(evutil_socket_t sig, short events, void *arg)
{
    printf("\nReceived SIGHUP, reloading configuration...\n");
    // reload_config();
}

int main(void)
{
    struct event_base *base = event_base_new();

    // 注册多个信号事件
    struct event *ev_sigint  = evsignal_new(base, SIGINT,  sigint_cb,  base);
    struct event *ev_sigterm = evsignal_new(base, SIGTERM, sigterm_cb, base);
    struct event *ev_sigusr1 = evsignal_new(base, SIGUSR1, sigusr1_cb, base);
    struct event *ev_sighup  = evsignal_new(base, SIGHUP,  sighup_cb,  base);

    event_add(ev_sigint,  NULL);
    event_add(ev_sigterm, NULL);
    event_add(ev_sigusr1, NULL);
    event_add(ev_sighup,  NULL);

    // ... 注册其他业务事件 ...

    printf("Server running. PID=%d\n", getpid());
    printf("Send SIGINT/SIGTERM to stop, SIGUSR1 for stats, SIGHUP to reload.\n");

    event_base_dispatch(base);

    // 清理
    event_free(ev_sigint);
    event_free(ev_sigterm);
    event_free(ev_sigusr1);
    event_free(ev_sighup);
    event_base_free(base);

    printf("Shutdown complete.\n");
    return 0;
}
```

### 5.2 只处理一次信号

```c
// 不使用 EV_PERSIST，信号触发一次后自动失效
struct event *ev = event_new(base, SIGINT, EV_SIGNAL,  // 不加 EV_PERSIST
                              one_shot_cb, NULL);
event_add(ev, NULL);
// 第一次 SIGINT → 调用 one_shot_cb，然后事件自动删除
// 之后的 SIGINT 使用默认行为（终止进程）
```

### 5.3 多信号共用一个回调

```c
static void any_signal_cb(evutil_socket_t sig, short events, void *arg)
{
    // 通过 sig 参数区分是哪个信号
    switch ((int)sig) {
    case SIGINT:  printf("Got SIGINT\n");  break;
    case SIGTERM: printf("Got SIGTERM\n"); break;
    case SIGUSR1: printf("Got SIGUSR1\n"); break;
    }
}

// 传入 event_self_cbarg()，让回调参数是 event 本身
struct event *ev_int  = evsignal_new(base, SIGINT,  any_signal_cb, NULL);
struct event *ev_term = evsignal_new(base, SIGTERM, any_signal_cb, NULL);
struct event *ev_usr1 = evsignal_new(base, SIGUSR1, any_signal_cb, NULL);
```

## 6. 全局配置：日志、内存、调试

### 6.1 自定义日志（`event.h` 第 734~808 行）

libevent 内部产生的所有日志默认打印到 stderr，可以重定向到自己的日志系统：

```c
// 日志级别
#define EVENT_LOG_DEBUG  0   // 调试（详细）
#define EVENT_LOG_MSG    1   // 普通信息
#define EVENT_LOG_WARN   2   // 警告
#define EVENT_LOG_ERR    3   // 错误

// 设置日志回调（全局，线程不安全，必须在创建 event_base 前调用）
void event_set_log_callback(event_log_cb cb);

typedef void (*event_log_cb)(int severity, const char *msg);
```

**实战**：接入自定义日志系统

```c
#include <event2/event.h>
#include <time.h>

static void my_log_cb(int severity, const char *msg)
{
    const char *level_str;
    switch (severity) {
    case EVENT_LOG_DEBUG: level_str = "DEBUG"; break;
    case EVENT_LOG_MSG:   level_str = "INFO";  break;
    case EVENT_LOG_WARN:  level_str = "WARN";  break;
    case EVENT_LOG_ERR:   level_str = "ERROR"; break;
    default:              level_str = "?????"; break;
    }

    time_t t = time(NULL);
    struct tm tm_info;
    localtime_r(&t, &tm_info);
    char timebuf[20];
    strftime(timebuf, sizeof(timebuf), "%Y-%m-%d %H:%M:%S", &tm_info);

    fprintf(stderr, "[%s] [libevent] [%s] %s\n", timebuf, level_str, msg);

    // 或者写入日志文件、发给 syslog、写入 Kafka...
}

int main(void) {
    // 必须在 event_base_new() 之前设置
    event_set_log_callback(my_log_cb);

    struct event_base *base = event_base_new();
    // ...
}
```

**注意**：日志回调函数内部**绝对不能**调用任何 libevent 函数，否则可能重入死锁。

### 6.2 开启调试日志

```c
// 打开所有调试日志（非常详细，仅用于排查问题）
// 必须在任何 event_base 操作之前调用
event_enable_debug_logging(EVENT_DBG_ALL);

// 关闭调试日志
event_enable_debug_logging(EVENT_DBG_NONE);
```

调试日志会输出每次 event_add / event_del / callback 调用的详细信息，是排查 libevent 问题的利器。

### 6.3 致命错误处理

```c
// 默认行为：libevent 遇到致命错误时调用 exit(1)
// 自定义处理：可以记录崩溃信息、保存状态、发送告警

typedef void (*event_fatal_cb)(int err);
void event_set_fatal_callback(event_fatal_cb cb);

static void my_fatal_cb(int err) {
    fprintf(stderr, "FATAL: libevent internal error %d\n", err);
    // 记录崩溃堆栈
    // abort()；或者尝试保存状态
    abort();  // 确保生成 core dump
}

// 注意：fatal callback 被调用时，libevent 状态已经不可信
// 调用后任何 libevent 操作都可能产生未定义行为
event_set_fatal_callback(my_fatal_cb);
```

### 6.4 自定义内存分配器

```c
// 替换 libevent 内部的 malloc/realloc/free
// 常用场景：内存池、内存泄漏检测、jemalloc/tcmalloc

void event_set_mem_functions(
    void *(*malloc_fn)(size_t sz),
    void *(*realloc_fn)(void *ptr, size_t sz),
    void (*free_fn)(void *ptr)
);
```

**重要约束**：
- 必须在**任何** libevent 函数调用之前设置
- 替换后，所有 libevent 分配的内存都来自新分配器
- 不能混用旧/新分配器释放内存

**接入 jemalloc 示例**：

```c
#include <jemalloc/jemalloc.h>
#include <event2/event.h>

int main(void) {
    // 使用 jemalloc 替换 libevent 的内存分配
    event_set_mem_functions(je_malloc, je_realloc, je_free);

    struct event_base *base = event_base_new();
    // 所有 libevent 内部分配都走 jemalloc
    // ...
}
```

**接入内存泄漏检测**：

```c
static size_t g_alloc_count = 0;
static size_t g_alloc_bytes = 0;

static void *debug_malloc(size_t sz) {
    g_alloc_count++;
    g_alloc_bytes += sz;
    return malloc(sz);
}

static void *debug_realloc(void *ptr, size_t sz) {
    return realloc(ptr, sz);
}

static void debug_free(void *ptr) {
    g_alloc_count--;
    free(ptr);
}

// 程序退出时检查是否有泄漏
atexit(check_leaks);
event_set_mem_functions(debug_malloc, debug_realloc, debug_free);
```

### 6.5 调试模式

```c
// 开启 libevent 调试模式：检测 double-add、use-after-free 等编程错误
// 必须在任何 event_base 操作之前调用
void event_enable_debug_mode(void);
```

调试模式的行为：
- 追踪每个 `event` 的分配位置（用于 `event_debug_unassign`）
- 检测 `event_add` 了已经 add 的 event（编程错误）
- 检测 `event_free` 了还在 active 状态的 event
- 有一定内存和性能开销，**仅用于开发测试，生产环境禁用**

```c
int main(void) {
    // 开发时启用所有调试
    event_enable_debug_mode();
    event_enable_debug_logging(EVENT_DBG_ALL);
    event_set_log_callback(my_log_cb);

    struct event_base *base = event_base_new();
    // ...
}
```

## 8. 综合实战：健壮的服务器信号处理框架

```c
// robust_server.c：完整的信号处理 + 日志配置框架

#include <event2/event.h>
#include <event2/bufferevent.h>
#include <event2/listener.h>
#include <signal.h>
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <time.h>

// ── 日志系统 ─────────────────────────────────────────────────────
static FILE *g_logfile = NULL;

static void my_log_cb(int severity, const char *msg)
{
    if (!g_logfile) return;

    const char *lv = severity == EVENT_LOG_DEBUG ? "DEBUG"
                   : severity == EVENT_LOG_MSG   ? "INFO"
                   : severity == EVENT_LOG_WARN  ? "WARN"
                   : "ERROR";

    time_t t = time(NULL);
    char timebuf[20];
    struct tm tm_info;
    localtime_r(&t, &tm_info);
    strftime(timebuf, sizeof(timebuf), "%H:%M:%S", &tm_info);

    fprintf(g_logfile, "[%s][%s] %s\n", timebuf, lv, msg);
    fflush(g_logfile);
}

// ── 服务器上下文 ─────────────────────────────────────────────────
struct server {
    struct event_base *base;
    struct evconnlistener *listener;
    struct event *sig_int;
    struct event *sig_term;
    struct event *sig_usr1;
    struct event *sig_hup;

    int active_conns;
    ev_uint64_t total_conns;
    ev_uint64_t total_bytes;

    int is_shutting_down;
};

static struct server g_srv = {0};

// ── 信号回调 ─────────────────────────────────────────────────────
static void print_stats(void)
{
    printf("Active: %d | Total: %llu | Bytes: %llu\n",
        g_srv.active_conns,
        (unsigned long long)g_srv.total_conns,
        (unsigned long long)g_srv.total_bytes);
}

static void sigint_cb(evutil_socket_t sig, short ev, void *arg)
{
    if (g_srv.is_shutting_down) return;
    g_srv.is_shutting_down = 1;

    printf("\n[SIGINT] Graceful shutdown in 3s...\n");
    print_stats();

    // 停止接受新连接
    evconnlistener_disable(g_srv.listener);

    // 3 秒后退出（给当前连接时间完成）
    struct timeval delay = {3, 0};
    event_base_loopexit(g_srv.base, &delay);
}

static void sigterm_cb(evutil_socket_t sig, short ev, void *arg)
{
    printf("\n[SIGTERM] Immediate shutdown.\n");
    print_stats();
    event_base_loopbreak(g_srv.base);
}

static void sigusr1_cb(evutil_socket_t sig, short ev, void *arg)
{
    printf("\n[SIGUSR1] Stats:\n");
    print_stats();
    // 继续运行
}

static void sighup_cb(evutil_socket_t sig, short ev, void *arg)
{
    printf("\n[SIGHUP] Reloading config...\n");
    // reload_config();
}

// ── 连接处理 ─────────────────────────────────────────────────────
static void conn_readcb(struct bufferevent *bev, void *ctx) {
    struct evbuffer *in  = bufferevent_get_input(bev);
    struct evbuffer *out = bufferevent_get_output(bev);
    g_srv.total_bytes += evbuffer_get_length(in);
    evbuffer_add_buffer(out, in);
}

static void conn_eventcb(struct bufferevent *bev, short events, void *ctx) {
    if (events & (BEV_EVENT_EOF | BEV_EVENT_ERROR | BEV_EVENT_TIMEOUT)) {
        --g_srv.active_conns;
        bufferevent_free(bev);
    }
}

static void accept_cb(struct evconnlistener *lev, evutil_socket_t fd,
                      struct sockaddr *addr, int socklen, void *ctx)
{
    ++g_srv.active_conns;
    ++g_srv.total_conns;

    struct bufferevent *bev = bufferevent_socket_new(
        g_srv.base, fd, BEV_OPT_CLOSE_ON_FREE);

    bufferevent_setcb(bev, conn_readcb, NULL, conn_eventcb, NULL);

    struct timeval tv = {60, 0};
    bufferevent_set_timeouts(bev, &tv, NULL);
    bufferevent_enable(bev, EV_READ | EV_WRITE);
}

// ── 主函数 ────────────────────────────────────────────────────────
int main(int argc, char **argv)
{
    // 1. 全局配置（必须在 event_base_new 之前）
    g_logfile = stderr;
    event_set_log_callback(my_log_cb);
    // 开发时启用：event_enable_debug_mode();
    // 开发时启用：event_enable_debug_logging(EVENT_DBG_ALL);

    // 2. 创建事件循环
    g_srv.base = event_base_new();

    // 3. 注册信号事件
    g_srv.sig_int  = evsignal_new(g_srv.base, SIGINT,  sigint_cb,  NULL);
    g_srv.sig_term = evsignal_new(g_srv.base, SIGTERM, sigterm_cb, NULL);
    g_srv.sig_usr1 = evsignal_new(g_srv.base, SIGUSR1, sigusr1_cb, NULL);
    g_srv.sig_hup  = evsignal_new(g_srv.base, SIGHUP,  sighup_cb,  NULL);

    event_add(g_srv.sig_int,  NULL);
    event_add(g_srv.sig_term, NULL);
    event_add(g_srv.sig_usr1, NULL);
    event_add(g_srv.sig_hup,  NULL);

    // 4. 创建监听器
    struct sockaddr_in sin = {0};
    sin.sin_family = AF_INET;
    sin.sin_port   = htons(9999);

    g_srv.listener = evconnlistener_new_bind(
        g_srv.base, accept_cb, NULL,
        LEV_OPT_REUSEABLE | LEV_OPT_CLOSE_ON_FREE, -1,
        (struct sockaddr *)&sin, sizeof(sin));

    printf("Server PID=%d listening on :9999 (backend=%s)\n",
        getpid(), event_base_get_method(g_srv.base));
    printf("Signals: SIGINT=graceful, SIGTERM=immediate, "
           "SIGUSR1=stats, SIGHUP=reload\n");

    // 5. 运行事件循环
    event_base_dispatch(g_srv.base);

    // 6. 清理
    event_free(g_srv.sig_int);
    event_free(g_srv.sig_term);
    event_free(g_srv.sig_usr1);
    event_free(g_srv.sig_hup);
    evconnlistener_free(g_srv.listener);
    event_base_free(g_srv.base);

    printf("Shutdown complete. Final stats:\n");
    print_stats();
    return 0;
}
```

## 9. 信号处理的常见陷阱

### 9.1 SIGPIPE：被遗忘的信号

```c
// 写入已关闭的 socket 会产生 SIGPIPE，默认行为是终止进程！
// 必须在程序启动时忽略它

// 方法 1：用 signal() 忽略（简单）
signal(SIGPIPE, SIG_IGN);

// 方法 2：用 sigaction() 忽略（更规范）
struct sigaction sa = {0};
sa.sa_handler = SIG_IGN;
sigaction(SIGPIPE, &sa, NULL);

// 方法 3：在每个 write/send 时加 MSG_NOSIGNAL（不可移植）
send(fd, data, len, MSG_NOSIGNAL);
```

libevent 本身不会帮你处理 SIGPIPE，必须自己处理。

### 9.2 信号与多线程

```c
// 问题：只有一个 event_base 能接收信号
// 解决：在主线程（main thread）的 base 上注册信号

// 正确的多线程架构：
// 主线程：event_base_main + 信号处理
// 工作线程：event_base_worker（不注册信号）

// 错误：在 worker base 上注册信号（不可靠）
evsignal_new(worker_base, SIGINT, ...);  // 不推荐！
```

### 9.3 不要在信号回调里做阻塞操作

```c
// 错误：信号回调在事件循环中执行，阻塞会影响所有事件
static void sighup_cb(evutil_socket_t sig, short ev, void *arg) {
    // 阻塞操作！不应该在这里做
    sleep(5);  // 阻塞 5 秒，所有其他事件都停摆
    load_config_from_database();  // 可能很慢
}

// 正确：仅设置标志，在下一轮循环中处理
static volatile int g_reload_config = 0;
static void sighup_cb(evutil_socket_t sig, short ev, void *arg) {
    g_reload_config = 1;  // 仅设置标志
}
// 在定时器或其他合适的地方检查 g_reload_config
```

## 11. 本课小结

| 概念 | 关键点 |
|------|--------|
| 管道技巧 | 信号 → write(pipe) → epoll_wait → evsig_cb → 用户回调 |
| `evsig_handler` | 只能调用 async-signal-safe 函数，仅写 1 字节到管道 |
| signalfd | Linux 专属，每个信号一个 fd，更干净，无全局 base 限制 |
| `evsignal_new` | 宏，等价于 `event_new(base, sig, EV_SIGNAL|EV_PERSIST, cb, arg)` |
| 单 base 限制 | 同一时刻只有一个 base 能用管道技巧接收信号（signalfd 无此限制）|
| `event_set_log_callback` | 重定向 libevent 日志，回调内不能调用 libevent 函数 |
| `event_set_mem_functions` | 替换内存分配器，必须在任何 libevent 操作前调用 |
| `event_enable_debug_mode` | 开发调试用，生产禁用 |
| SIGPIPE | libevent 不处理，需手动 `signal(SIGPIPE, SIG_IGN)` |

**三句话总结**：

1. 信号处理的核心是管道技巧：signal handler 只写一字节到 pipe，事件循环读取后在安全环境中调用用户回调
2. `evsignal_new` = `event_new` + `EV_SIGNAL|EV_PERSIST`，信号事件几乎总是持久的
3. 生产服务器必备三件套：`signal(SIGPIPE, SIG_IGN)` + `evsignal_new(SIGTERM)` + `evsignal_new(SIGHUP)`

## 附：练习 3 答案

1. **async-signal-safe 要求**：
   信号可以在任意时刻打断任意代码执行。如果在 `printf` 内部（持有某个锁）被信号打断，信号 handler 再次调用 `printf`（尝试获取同一把锁），就会**死锁**。`write(2)` 是 POSIX 明确标注的 async-signal-safe 函数，不使用任何用户级锁，因此安全。

2. **去掉 EV_PERSIST 的效果**：
   信号触发一次后事件自动从 base 中删除，后续收到的相同信号使用操作系统的默认处理方式（如 SIGINT 默认终止进程）。这意味着应用程序只能处理该信号一次，对于需要持续处理信号的服务器来说通常是错误的。只有在"只响应一次信号"的场景下才去掉 EV_PERSIST。

3. **signalfd 的优势**：
   - 无需 socketpair，减少 fd 数量
   - 每个信号独立 fd，无需全局变量 `evsig_base`，多个 event_base 可独立接收各自的信号
   - 可以读取 `signalfd_siginfo` 获得信号发送者的 PID/UID 等详细信息
   - libevent 在编译时检测到系统有 `<sys/signalfd.h>`（Linux 2.6.22+）且启用了该特性时自动使用 signalfd；否则回退到 socketpair 技巧
